Appendix D — Reproducibility

E Reproducibility

This appendix documents the software environment, configuration parameters, data processing steps, and analysis scripts required to reproduce all results presented in this report.

E.1 Software Environment

All analyses were conducted using bayesRVE version 0.6.0 running on the following platform.

Operating system. macOS (ARM64, Apple Silicon).

R and core packages. Table E.1 lists the R packages and minimum versions used throughout the report.

CmdStan. All Stan models were compiled and sampled via CmdStan (development version installed from stan-dev/cmdstanr). Three Stan programs ship with the package:

  • bm_ce_ncp.stan — Correlated-effects model, non-centered parameterization (default).
  • bm_ce_cp.stan — Correlated-effects model, centered parameterization.
  • bm_che_ncp.stan — Correlated-hierarchical-effects model, non-centered parameterization.

Stan models are located in bayesRVE/inst/stan/ and are compiled at first use via cmdstanr::cmdstan_model().

E.2 Random Seeds and MCMC Configuration

All MCMC analyses used the following configuration, set explicitly in every analysis script.

Seeds are passed to bm_fit() via the seed argument, which forwards to cmdstanr::sample(). No implicit random number generation occurs outside of Stan sampling. The elevated adapt_delta (0.95, versus the Stan default of 0.80) reduces the risk of divergent transitions in hierarchical models.

Priors. All analyses used the following weakly informative priors:

  • Regression coefficients: \(\beta_k \sim \text{Normal}(0, 10)\)
  • Between-study SD (CE and CHE): \(\tau \sim \text{half-Cauchy}(0, 0.5)\)
  • Within-study SD (CHE only): \(\omega \sim \text{half-Cauchy}(0, 0.5)\)

These priors are set via bm_prior() and passed to bm_fit().

E.3 Data Processing Notes

Source dataset. The empirical application uses the Tanner-Smith & Lipsey (2015) dataset of brief alcohol interventions for adolescents and young adults, as reanalyzed by Pustejovsky & Tipton (2022). The dataset is stored as an RDS file at:

dev/manuscript/data/Tanner-Smith-Lipsey-2015-subset.rds

Analytic subset. Records with missing values on percoll, attrition_all, or permale are excluded, yielding:

  • \(J = 107\) studies
  • \(N = 1{,}198\) effect sizes

Effect size metric. Standardized mean difference (Hedges’ \(g\)), with sampling variances reported in the original dataset (variable V).

Imputed covariance matrix. The block-diagonal working variance-covariance matrix is constructed via metafor::impute_covariance_matrix() with \(\rho = 0.6\) and smooth_vi = TRUE.

Moderator centering. All continuous moderators are centered prior to model fitting. Table E.3 documents the transformations applied.

Categorical moderators. dv_cat (6 outcome categories) and Ctype (4 control-group types) are converted to unordered factors. All models use cell-mean coding (no intercept; formula prefix 0 +).

E.4 Chapter-to-Script Map

Table E.4 maps each chapter of the report to the analysis scripts and key output files that produced its results.

The primary results object v060_results.rds (607 MB) contains all fitted model objects, sandwich variance objects, CCC calibration objects, MCMC draws for variance components, and pre-computed summary tables. This file is generated by running 00_full_analysis.R end-to-end with the MCMC settings documented in Table E.2.

Reproduction procedure. To reproduce the empirical results from a clean state:

  1. Install bayesRVE v0.6.0 and all dependencies listed in Table E.1.
  2. Ensure CmdStan is installed and accessible via cmdstanr::cmdstan_path().
  3. Place the TSL15 dataset at the path specified above.
  4. Execute 00_full_analysis.R. Runtime is approximately 20–40 minutes depending on hardware (ARM64 Mac: ~25 minutes).
  5. The script writes v060_results.rds and CSV summary tables to the output directory.

E.5 Test Suite Summary

The bayesRVE package includes a comprehensive test suite executed via testthat (edition 3). Table E.5 summarizes the current results as of v0.6.0.

Tests are organized into groups targeting distinct validation layers. Table E.6 documents the major groups.

Tests can be executed from the package root with:

devtools::test("bayesRVE")

The D25 plug-in parity tests inject REML point estimates for \(\hat\tau^2\) (and \(\hat\omega^2\) for CHE) into the bayesRVE sandwich machinery and verify that the resulting \(V_\text{CR2}\) matrix matches the clubSandwich reference implementation at a tolerance of \(10^{-6}\). This isolates the linear algebra from MCMC sampling noise and confirms correctness of the CR2 implementation.

## Data Processing Notes {#sec-repro-data}

**Source dataset.** The empirical application uses the @tannersmith2015 dataset of brief alcohol interventions for adolescents and young adults, as reanalyzed by @pustejovsky2022. The dataset is stored as an RDS file at:

```
dev/manuscript/data/Tanner-Smith-Lipsey-2015-subset.rds
```

**Analytic subset.** Records with missing values on `percoll`, `attrition_all`, or `permale` are excluded, yielding:

- $J = 107$ studies
- $N = 1{,}198$ effect sizes

**Effect size metric.** Standardized mean difference (Hedges' $g$), with sampling variances reported in the original dataset (variable `V`).

**Imputed covariance matrix.** The block-diagonal working variance-covariance matrix is constructed via `metafor::impute_covariance_matrix()` with $\rho = 0.6$ and `smooth_vi = TRUE`.

**Moderator centering.** All continuous moderators are centered prior to model fitting. @tbl-repro-center documents the transformations applied.

| Variable | Centering |
|:---------|:----------|
| `postwks_c` | $\min(\texttt{postwks}, 26) - 12$ |
| `postwks_long` | Indicator: $\texttt{postwks} > 26$ |
| `percoll_c` | $\texttt{percoll} - 1$ |
| `permale_c` | $\texttt{permale} - 0.5$ |
| `attrition_c` | $\texttt{attrition\_all} - \text{median}(\texttt{attrition\_all})$ |

: Moderator centering transformations applied to the TSL15 dataset. {#tbl-repro-center}

**Categorical moderators.** `dv_cat` (6 outcome categories) and `Ctype` (4 control-group types) are converted to unordered factors. All models use cell-mean coding (no intercept; formula prefix `0 +`).
